## 出力先欄の状態

- 未設定（空文字列）: 赤枠（`border: 2px solid red`）でハイライト。バッチ実行ボタンは無効化
- 設定済み: 通常スタイル。`[📁]` ボタンでフォルダダイアログを開く

---

## 主要操作フロー

1. **新規作成**: 「新規」→ 空のプロジェクト → シーン/オブジェクトを追加
2. **既存開く**: 「開く」→ ファイルダイアログ（`.yaml` / `.yml`）→ 読み込み
3. **プレビュー**: ▶ で再生（Canvas 上でオブジェクトをドラッグして位置を調整可能）
4. **書き出し**: 「書き出し」→ バリデーション → 進捗ダイアログ → 完了/エラー

---

## ファイル指定

- ファイルパス入力欄 + 「参照...」ボタン（Tauriのネイティブファイルダイアログ）
- ドラッグ&ドロップ: ファイルを入力欄にドロップしてパスを設定（`dragDropEnabled` を `false` に設定して Tauri のグローバル drag-drop イベントを無効化し、HTML5 標準の `dragover` / `drop` イベントのみ使用する。`tauri.conf.json` の `windows[0].dragDropEnabled: false` を設定する）

## キーボードショートカット

| キー | 操作 |
|------|------|
| `Cmd/Ctrl + N` | 新規プロジェクト |
| `Cmd/Ctrl + O` | 開く |
| `Cmd/Ctrl + S` | 上書き保存 |
| `Cmd/Ctrl + Shift + S` | 名前を付けて保存 |
| `Space` | プレビュー再生/停止 |
| `Escape` | プレビュー再生中なら**停止を優先**、停止中なら選択解除 |
| `Delete` / `Backspace` | 選択中のオブジェクト・エントリを削除（確認ダイアログなし） |
| `Cmd/Ctrl + D` | 選択中要素を複製（優先順位: 選択オブジェクト > 選択シーン > 無効化） |
| `←` / `→`（一時停止中、非テキスト入力時） | 1フレーム（1/fps 秒）移動 |
| `←` / `→`（再生中、非テキスト入力時） | 5秒シーク |

矢印キー動作の対象判定:
- プレビュー領域（Canvas）・シークバー・タイムラインのいずれかがフォーカスを持つときに有効
- テキスト入力・数値入力フィールドにフォーカスがある場合はそれらの通常動作（文字カーソル移動等）を優先

## エディタUX詳細

### Undo/Redo

初期バージョンでは非対応（実装コストに対してスコープが大きいためv2以降）。

### 未保存変更の扱い

- 未保存の変更がある状態で「新規作成」「開く」「ウィンドウを閉じる」を行うと確認ダイアログを表示する
- ダイアログ: 「変更が保存されていません。保存しますか？ [保存して続ける] [保存しないで続ける] [キャンセル]」

### 自動保存

初期バージョンでは非対応（シンプルさ優先）。明示的な保存操作のみ。

### プレビュー再生中の編集

プロパティパネルで属性を編集した時点でプレビューを**自動停止**し、変更を反映。再生の再開はユーザー操作で行う。

## オブジェクト操作

- **選択**: タイムライン上のオブジェクトブロックをクリック / Canvas上のオブジェクトをクリック（Z順最前面優先）
- **削除**: 選択後に `Delete` / `Backspace` キー、またはコンテキストメニューから「削除」
- **Z順変更**: プロパティパネルの「前面へ」「背面へ」ボタン（YAML内の配列順を移動）
- **選択中ハイライト**: Canvasに半透明の青枠（2px、rgba(0,128,255,0.7)）を描画

### 新規オブジェクト追加フロー

1. タイムラインの「+ オブジェクト追加」ボタンをクリック
2. タイプ選択ドロップダウンが表示（video / image / audio / text）
3. video / image / audio: 「固定（variable: false）」か「可変（variable: true）」を選択するステップを挟み、固定の場合は即座にファイルダイアログを開く
4. text: テキスト入力フィールドを持つ簡易ダイアログを表示（固定 / 可変を選択可能）
5. 確定後、以下のデフォルト値でオブジェクトをシーンに追加

### デフォルト値

| 種別 | width × height | x, y | start | duration | opacity | volume |
|------|---------------|------|-------|----------|---------|--------|
| video | 出力解像度（例: 1920×1080） | (0, 0) | 0.0 | — | 100 | 100 |
| image | min(元サイズ, 出力解像度) | (0, 0) | 0.0 | 0.0 | 100 | — |
| text | 400 × 60 | (0, 0) | 0.0 | 0.0 | 100 | — |
| audio | — | — | 0.0 | 0.0 | — | 100 |

**テキストオブジェクト固有のデフォルト値:**

| 属性 | 初期値 |
|------|-------|
| `font` | `NotoSansCJK-Regular` |
| `font_size` | `24`（pt） |
| `color` | `#ffffff` |
| `align` | `left` |
| `background_color` | 未設定（背景矩形なし） |

**音声オブジェクト固有のデフォルト値:**

| 属性 | 初期値 |
|------|-------|
| `loop` | `loop`（Rust `#[default] Loop` と整合） |
| `fade_in` / `fade_out` | 未設定（0秒扱い） |

### Canvas 上のドラッグ移動

- 選択中オブジェクトをドラッグで移動（`x`, `y` を更新）✅ 実装済み
- ドラッグ開始時にオブジェクトを選択状態にする
- 移動中はプロパティパネルの値をリアルタイム更新する
- リサイズハンドル（8点）は v2 以降
- Canvas 上でのテキスト直接インライン編集は v2 以降

## 新規プロジェクトの初期状態

完全空（シーン0件）。GUI上には「シーンを追加」のプレースホルダを表示。

`output_folder` の初期値は空文字列 `""`。バリデーションで「出力フォルダが設定されていません」エラーとして書き出しをブロックする。GUI のツールバーに出力フォルダ設定欄を常時表示し、未設定時は赤枠でハイライトする。

`output_folder` は**絶対パスのみ**受け付ける。

---

## 出力設定のGUI

**CRF 入力方法:**
- `CRF`: 数値入力フィールド + スライダー併用（h264/h265: 0–51、vp9: 0–63、整数）。現在の推奨値をヒントテキストで表示（例: 「H.264 推奨: 23（18–28）」）
- `preset`: セレクトボックス（`ultrafast` / `superfast` / `veryfast` / `faster` / `fast` / `medium` / `slow` / `slower` / `veryslow`）。VP9 選択時はグレーアウト

## エラーメッセージ設計

### エラー分類別のメッセージ方針

| 分類 | メッセージ形式（日本語） |
|-----|----------------------|
| YAMLパース | 「プロジェクトファイルを読み込めませんでした。（行N: {serde_ymlのエラー詳細}）」 |
| スキーマ検証 | 「設定に問題があります。（{scene.id} > {object.id}）: {具体的な問題}」 |
| ファイルI/O | 「ファイルが見つかりません: {path}」 |
| FFmpegエラー | 「エンコードエラー（{entry_name}）: {短い概要} ▶ 詳細を表示（stderr原文）」 |
| 内部バグ | 「予期しないエラーが発生しました。ログファイルをご確認ください: {log_path}」 |

- ユーザー向けメッセージは**日本語**
- FFmpegのstderr原文は折りたたみ（「▶ 詳細を表示」）で併記

### YAMLパースエラーの行番号表示

`serde_yml` の `Error::location()` で行番号を取得し、エラーメッセージに「行N列M」形式で表示。`location()` は `Option<Location>` を返すため、`None` の場合は「行番号不明」と表示し、`serde_yml` のエラー詳細文字列のみを提示する。

## 起動時画面

`recent_files` が1件以上あれば Recent Files 一覧を**エディタ内のモーダルダイアログ**として表示する。なければ空エディタを直接表示する。

**モーダルのキャンセル動作**: ESC キー・「×」ボタン・背景クリックによる閉じ操作は**空エディタ状態への遷移**として扱う。

**ESC キーの優先順位**: モーダル表示中は Svelte の `onkeydown` イベントハンドラをモーダルコンポーネント側でトラップし（`event.stopPropagation()`）、モーダルを閉じる動作を優先する。

## タイトルバーの表示形式

- `ezmm — {ファイル名}`（保存済み）
- `ezmm — {ファイル名}*`（ファイル名あり・未保存変更あり）
- `ezmm — 無題`（新規作成直後・未変更）
- `ezmm — 無題*`（新規作成後に編集あり）

`*` は最初の編集操作を行った時点で付与する。新規作成直後（まだ何も変更していない状態）は `*` を付けない。
