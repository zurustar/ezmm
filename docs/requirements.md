# 要件定義

## 概要

社内向けの動画編集ツール。  

## ユーザー

- **主なユーザー**: 社内映像担当者・広報担当者
- 操作がシンプルなため、結果的に誰でも使える想定

---

## コアコンセプト

### プロジェクト

すべての設定を1つの **YAMLファイル** に収める。GUIで視覚的に編集し、ファイルとして保存・読み込みする。

プロジェクトファイルの構成：

| セクション | 内容 |
|-----------|------|
| シーン定義 | タイムラインのシーン構成とオブジェクト配置（映像・画像・テキスト・音声） |
| 出力設定 | 出力ファイル名・解像度・フレームレート・コーデック・形式 |
| 出力フォルダ | 完成動画の保存先 |

### シーン

タイムラインは複数の**シーン**で構成される。シーンは定義された順に連続して再生される。

**シーンの長さ：**

シーン内の全オブジェクトの終了時刻の最大値がシーンの長さになる。

- **映像オブジェクト**の終了時刻 = 表示開始時間 ＋ (トリム終了 − トリム開始)。トリム未設定時はファイル全体の長さを使う
- **画像・テキスト・音声オブジェクト**の終了時刻 = 表示開始時間 ＋ 表示時間
- 表示時間=0（シーン終端まで）のオブジェクトは、上記で決まったシーン長全体に渡って表示される
- 明示的な時間を持つオブジェクトが1つも存在しない場合は、シーンに固定秒数を直接指定する

### オブジェクト

各シーンには複数のオブジェクトを配置できる。同じ時刻に複数のオブジェクトが存在する場合は重ね合わせて合成される。オブジェクトの時間はシーン先頭からの秒数で指定する。

| 種別 | 説明 |
|------|------|
| **映像** | 動画ファイルを指定した位置・タイミングで再生 |
| **画像** | 画像ファイルを指定した位置・タイミングで表示 |
| **テキスト** | 文字列を指定した位置・タイミングで表示 |
| **音声** | 音声ファイルを指定したタイミングで再生（視覚的な位置・サイズは持たない） |

各オブジェクトの属性：

| 属性 | 対象 | 内容 |
|------|------|------|
| id | 全 | オブジェクトの識別子 |
| type | 全 | オブジェクトの種別: `video` / `image` / `text` / `audio` |
| ファイル | 映像・画像・音声 | ファイルのパス |
| テキスト内容 | テキスト | 表示する文字列 |
| 表示位置 | 映像・画像・テキスト | 画面上の X / Y 座標 |
| サイズ | 映像・画像・テキスト | 幅・高さ |
| 表示開始時間 | 全 | シーン先頭からの秒数 |
| 表示時間 | 画像・テキスト・音声 | 表示/再生し続ける秒数（0でシーン終端まで） |
| トリム開始 | 映像 | ソースファイルの再生開始位置（秒）。省略時は 0 |
| トリム終了 | 映像 | ソースファイルの再生終了位置（秒）。省略時はファイル末尾まで |
| 透明度 | 映像・画像・テキスト | 0〜100% |
| 音量 | 映像・音声 | 音声の音量（0〜100%） |
| フェードイン | 音声 | 再生開始時のフェードイン秒数 |
| フェードアウト | 音声 | 再生終了時のフェードアウト秒数 |
| ループ | 音声 | 音声ファイルが表示時間より短い場合の挙動: `loop`（繰り返し）または `silence`（無音） |
| フォント・文字色・背景色 | テキスト | テキスト装飾 |

**トリム・中間カット**: トリム開始/終了で1本の映像オブジェクトの使用範囲を指定する。動画の中間部分をカットしたい場合は、同じファイルを参照する映像オブジェクトを2つ並べ、それぞれ異なるトリム範囲を設定する（例: オブジェクト1 で 0〜5秒、オブジェクト2 で 8秒〜末尾を使うと、5〜8秒が除去される）。

**レイヤー順序（Z順）**: YAML内の記述順で決まる。後に記述されたオブジェクトが上（前面）に表示される。

### 出力設定

プロジェクトに出力設定を保持する。

| 設定 | 内容 |
|------|------|
| 出力ファイル名 | 生成される動画ファイルの名前（拡張子なし） |
| 解像度 | 出力動画の幅・高さ（例: 1920×1080） |
| フレームレート | 出力動画のフレームレート（例: 30fps） |
| コーデック | 映像コーデック（例: H.264） |
| ファイル形式 | 出力ファイルの拡張子（例: mp4） |

## 機能要件

### 1. プロジェクトエディタ（GUI）

- プロジェクトファイル（YAML）の作成・保存・読み込みができる
- シーンの追加・削除・並び替えができる。シーンには任意の名前をつけられる（未設定時は「シーン 1」「シーン 2」のように連番表示）
- シーン内に動画・画像・テキスト・音声を追加・編集・削除できる。各オブジェクトにも任意の名前をつけられる（未設定時はファイル名やテキスト内容から自動生成）
- 各オブジェクトの属性（名前・位置・サイズ・タイミング・装飾など）をGUIで設定できる
- 出力設定（出力ファイル名・解像度・フレームレート・コーデック・形式）をGUIで編集できる
- 出力フォルダをGUIで指定できる

**アスペクト比の維持：**

- 映像・画像オブジェクトのサイズ（幅・高さ）は常にアスペクト比を連動して変更する。片方を変更すると、ソースファイルのアスペクト比を維持した値にもう片方が自動で更新される
- 出力設定の解像度も同様に、幅・高さを連動して変更する
- ファイルを読み込んだ際は、ソースファイルの自然な解像度（ffprobeで取得）からアスペクト比を確定する。その後の編集はこの値を基準にする
- ファイル読み込み時、映像・画像オブジェクトのサイズは出力キャンバスに収まるよう自動フィットし中央配置する

### 2. 書き出し

- プロジェクトを1本の動画ファイルとして書き出す
- 書き出し進捗をプログレスバーで表示する
- キャンセルができる
- エラーが発生した場合はエラー内容を表示する

### 3. プレビュー

- プロジェクトの内容をGUI上でリアルタイムに合成・再生プレビューできる
- FFmpegによる事前レンダリングなしに再生する
- 動画・音声オブジェクトの音声もプレビュー再生される（ユーザーがプレビュー画面をクリックして有効化）
- プレビューキャンバスはメインウインドウの利用可能な領域に収まるよう自動縮小される（アスペクト比を維持）
- 動画オブジェクトは再生が終了したら最終フレームを残さず非表示になる
- プレビューキャンバス上でオブジェクト（映像・画像・テキスト）をマウスでドラッグして位置（X/Y）を変更できる。ドラッグ中はリアルタイムにプレビューへ反映される。ドラッグ開始時に対象オブジェクトを選択状態にする

### 4. 動作環境

- デスクトップアプリ（Mac / Windows）

### 5. 対応フォーマット

入力ファイル（映像・画像・音声）はFFmpegが対応するすべての形式を利用できる。
