# 要件定義

## 概要

社内向けの動画バッチ編集ツール。  
役員メッセージなど、複数の人物が個別に撮影した動画に対して、同じ編集設定を適用し、一括で完成動画を生成する。

## ユーザー

- **主なユーザー**: 社内映像担当者・広報担当者
- 操作がシンプルなため、結果的に誰でも使える想定

---

## コアコンセプト

### プロジェクト

すべての設定を1つの **YAMLファイル** に収める。GUIで視覚的に編集し、ファイルとして保存・読み込みする。

プロジェクトファイルの構成：

| セクション | 内容 |
|-----------|------|
| シーン定義 | タイムラインのシーン構成とオブジェクト配置（映像・画像・テキスト・音声） |
| 出力設定 | 解像度・フレームレート・コーデック・形式 |
| 出力フォルダ | 完成動画の保存先 |
| エントリリスト | バッチ処理する動画のリスト（各エントリに可変値を指定） |

### シーン

タイムラインは複数の**シーン**で構成される。シーンは定義された順に連続して再生される。

**シーンの長さ：**

シーン内の全オブジェクトの終了時刻の最大値がシーンの長さになる。

- **映像オブジェクト**の終了時刻 = 表示開始時間 ＋ 動画ファイルの長さ（トリム後）
- **画像・テキスト・音声オブジェクト**の終了時刻 = 表示開始時間 ＋ 表示時間
- 表示時間=0（シーン終端まで）のオブジェクトは、上記で決まったシーン長全体に渡って表示される
- 明示的な時間を持つオブジェクトが1つも存在しない場合は、シーンに固定秒数を直接指定する

### オブジェクト

各シーンには複数のオブジェクトを配置できる。同じ時刻に複数のオブジェクトが存在する場合は重ね合わせて合成される。オブジェクトの時間はシーン先頭からの秒数で指定する。

| 種別 | 説明 |
|------|------|
| **映像** | 動画ファイルを指定した位置・タイミングで再生 |
| **画像** | 画像ファイルを指定した位置・タイミングで表示 |
| **テキスト** | 文字列を指定した位置・タイミングで表示 |
| **音声** | 音声ファイルを指定したタイミングで再生（視覚的な位置・サイズは持たない） |

各オブジェクトの属性：

| 属性 | 対象 | 内容 |
|------|------|------|
| id | 全 | オブジェクトの識別子。可変オブジェクトはエントリ側との対応付けに使用するため必須 |
| type | 全 | オブジェクトの種別: `video` / `image` / `text` / `audio` |
| variable | 全 | `true` にするとエントリごとに値を差し替え可能になる（省略時は固定） |
| ファイル | 映像・画像・音声 | 固定ファイルのパス（`variable: true` の場合はエントリ側で指定） |
| トリム | 映像（可変のみ） | 先頭・末尾のカット秒数（エントリごとに指定） |
| テキスト内容 | テキスト | 固定文字列、または「可変」指定（エントリごとに入力） |
| 表示位置 | 映像・画像・テキスト | 画面上の X / Y 座標 |
| サイズ | 映像・画像・テキスト | 幅・高さ |
| 表示開始時間 | 全 | シーン先頭からの秒数 |
| 表示時間 | 画像・テキスト・音声 | 表示/再生し続ける秒数（0でシーン終端まで） |
| 透明度 | 映像・画像・テキスト | 0〜100% |
| 音量 | 映像・音声 | 音声の音量（0〜100%） |
| フェードイン | 音声 | 再生開始時のフェードイン秒数 |
| フェードアウト | 音声 | 再生終了時のフェードアウト秒数 |
| ループ | 音声 | 音声ファイルが表示時間より短い場合の挙動: `loop`（繰り返し）または `silence`（無音） |
| フォント・文字色・背景色 | テキスト | テキスト装飾 |

すべてのオブジェクトは対等であり、映像・画像・テキスト・音声のいずれも「可変」にできる。

**レイヤー順序（Z順）**: YAML内の記述順で決まる。後に記述されたオブジェクトが上（前面）に表示される。

### エントリ

バッチ処理する1件分の動画。プロジェクトに複数のエントリを登録する。各エントリに指定する内容：
- **名前**（出力ファイル名に使用。例: `tanaka` → `tanaka.mp4`）
- 可変オブジェクトごとの値（映像ならファイル・トリム量、画像・音声ならファイル、テキストならテキスト内容）
- オブジェクトのidで対応付ける

### 出力設定

プロジェクトに出力設定を保持する。将来的にエントリごとの上書きに対応できるよう変数として持つ。

| 設定 | 内容 |
|------|------|
| 解像度 | 出力動画の幅・高さ（例: 1920×1080） |
| フレームレート | 出力動画のフレームレート（例: 30fps） |
| コーデック | 映像コーデック（例: H.264） |
| ファイル形式 | 出力ファイルの拡張子（例: mp4） |

## 機能要件

### 1. プロジェクトエディタ（GUI）

- プロジェクトファイル（YAML）の作成・保存・読み込みができる
- シーンの追加・削除・並び替えができる
- シーン内にオブジェクト（映像・画像・テキスト・音声）を追加・編集・削除できる
- 各オブジェクトの属性（id・位置・サイズ・タイミング・装飾など）をGUIで設定できる
- 出力設定（解像度・フレームレート・コーデック・形式）をGUIで編集できる
- エントリの追加・編集・削除ができる
- 各エントリに対して、可変オブジェクトの値をGUIで設定できる
- 出力フォルダをGUIで指定できる

### 2. バッチ実行

- プロジェクト内の全エントリを順次処理する
- 処理進捗を表示する（何件中何件完了か）
- エラーが発生した時点で処理を中断し、エラー内容を表示する

### 3. プレビュー

- 特定のエントリの処理結果を再生プレビューできる（1エントリ分）
- FFmpegによる事前レンダリングなしに、GUI上でリアルタイムに合成・再生する

### 4. 動作環境

- デスクトップアプリ（Mac / Windows）

### 5. 対応フォーマット

入力ファイル（映像・画像・音声）はFFmpegが対応するすべての形式を利用できる。
